# Documentação Técnica Definitiva do Banco de Dados (Oracle) — AgroGemini

Esta documentação consolida todo o conhecimento arquitetural, relacional e de segurança do banco de dados AgroGemini (v5 Expandida com Admin Analytics e Permissões).

---

## 1. Arquitetura e Estrutura de Dados (Total: 23 Tabelas)

O banco de dados é organizado em 6 camadas lógicas para garantir separação de preocupações e escalabilidade empresarial.

### 1.1 Camada de Identidade e Suporte (Base)
- `usuarios`: Cadastro central de pessoas. Tipos: UE (Externo), UP (Principal), UC (Comum), ADM.
- `enderecos`: Normalização de endereços com suporte a Lat/Long.
- `telefones_usuarios`: Contatos multi-canal.

### 1.2 Camada de Governança e Permissões (ACL)
- `permissoes_sistema`: Catálogo de funções do sistema (Ex: `LAUDO_EDITAR`, `FINANCEIRO_VER`).
- `usuario_permissoes`: Vínculo granular que permite `CONCEDER` ou `NEGAR` funções específicas a um usuário, independente do seu tipo.

### 1.3 Camada de Inteligência e Analytics (Admin)
- `admin_metricas_consolidadas`: Snapshots temporais de indicadores de negócio (MRR, Churn, Renovação, Crescimento).

### 1.4 Camada Agronômica (Business)
- `fazendas`: Unidades produtivas vinculadas a produtores.
- `talhoes`: Subdivisões operacionais com suporte espacial SIRGAS 2000.
- `fazenda_usuarios`: Vínculo N:M (Dono/Gerente).

### 1.5 Camada Laboratorial (Core Operations)
- `laboratorios`: Tenants operacionais com acreditação ISO17025.
- `amostras`: Registro de entrada e status de materiais para análise.
- `laudos`: Certificados de análise emitidos com hash de autenticação.
- `laudo_resultados`: Itens analíticos com classes interpretativas (NPK, etc).
- `importacoes`: Log de ingestão de dados CSV/XLSX.

### 1.6 Camada de Motor de Cálculo e Infraestrutura
- `configuracoes_calculo`: Definição de algoritmos e pipelines de ML.
- `variaveis_calculo`: Parâmetros dinâmicos para fórmulas.
- `limites_referencia`: Faixas interpretativas por bioma/cultura.
- `eventos_auditoria`: Trilha de auditoria JSON (Antes/Depois).
- `arquivos`: Repositório lógico de documentos e anexos.
- `laboratorio_usuarios`: Equipes vinculadas aos laboratórios.
- `telefones_laboratorios`: Contatos institucionais.
- `planos_assinaturas` / `assinaturas`: Controle de monetização e limites.

## 2. Segurança e Controle de Acesso

### 2.1 Modelo Híbrido RBAC + ACL
O sistema utiliza **RBAC** (Role Based Access Control) para permissões amplas por tipo de usuário e **ACL** (Access Control List) para bloqueios ou permissões "cirúrgicas" na tabela `usuario_permissoes`.

### 2.2 Virtual Private Database (VPD)
Isolamento multi-tenant garantido via `DBMS_RLS` no nível da conexão, utilizando o contexto `AGRO_CTX`.

---

## 3. Estratégia de Identidade (Oracle Identity)
Todas as tabelas utilizam a cláusula `GENERATED BY DEFAULT AS IDENTITY`, garantindo que o Oracle gerencie automaticamente o auto-incremento de chaves primárias com máxima performance.
