# Aprendizado sobre o Projeto AgroGemini

## Arquitetura do Sistema

### Backend (FastAPI + SQLAlchemy + Oracle)
- **Estrutura de Camadas:** O projeto segue uma estrutura clara: `Routers` -> `Services` -> `Repositories` -> `Models`.
- **Async por Padrão:** O backend é totalmente assíncrono, utilizando `AsyncSession` do SQLAlchemy para interagir com o Oracle.
- **Segurança:** Utiliza JWT para autenticação. As dependências de segurança (`deps.py`) gerenciam a extração do usuário e o controle de acesso baseado em papéis (`ADM`, `UP`, `UC`, `UE`).
- **Banco de Dados:** Utiliza Oracle com características específicas como `NUMBER GENERATED BY DEFAULT AS IDENTITY` e `TIMESTAMP WITH TIME ZONE`. A migração exige atenção a esses detalhes.

### Frontend (React + Vite + Framer Motion)
- **Estilização:** Usa uma mistura de CSS Vanilla com animações ricas via `Framer Motion` e ícones do `Lucide-React`.
- **Gestão de Estado:** Utiliza Context API (`LabContext.jsx`) para gerenciar dados globais do laboratório e do usuário, o que simplifica o acesso a dados de sessão em componentes profundos.
- **Internacionalização:** Sistema customizado em `locales.js`. Aprendizado: a falta de uma chave de tradução em um componente que não possui fallback pode derrubar toda a aplicação (Tela Branca).
- **Roteamento:** Usa roteamento baseado em Hash (`window.location.hash`), o que facilita o deploy em ambientes estáticos ou SPAs sem configuração complexa de servidor.

## Padrões Identificados
- **Normalização de Dados:** O frontend possui funções de normalização (`normalizeSample`, `normalizeStats`) no Contexto para garantir que dados vindos de diferentes versões da API ou estados do banco sejam exibidos de forma consistente.
- **Tematização Dinâmica:** O `useLabTheme.js` é um hook que centraliza as cores da aplicação, facilitando a manutenção e a alternância entre Dark/Light mode.

## Desafios e Soluções
- **Middleware de Autenticação:** A falha no `await` em funções assíncronas de dependência do FastAPI é um erro comum que resulta em objetos de coroutine sendo passados para os routers, quebrando o sistema silenciosamente (500 Internal Error).
- **Integração de Tipos:** A conversão entre os tipos de usuário do frontend (`producer`, `lab`) e as siglas do banco (`UE`, `UP`) exige mapeamento cuidadoso nos serviços de autenticação.

## Dicas para Manutenção Futura
- Sempre verificar se novas chaves de tradução foram adicionadas a todos os idiomas em `locales.js`.
- Ao modificar modelos, lembrar de atualizar os esquemas Pydantic correspondentes para evitar erros de validação no FastAPI.
- Manter o `LabContext` como a "fonte da verdade" para o estado do laboratório ativo.
